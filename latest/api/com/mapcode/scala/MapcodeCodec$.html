  6. def decode(mapcode: String, territoryContext: Territory.Territory): Point

    Decode a mapcode to a Point.

    Decode a mapcode to a Point. A reference territory is supplied for disambiguation (only used if applicable).

    The accepted format is: <mapcode> (note that a territory code is not allowed here)

    mapcode

    Mapcode.

    territoryContext

    Territory for disambiguation purposes.

    returns

    Point corresponding to mapcode. Latitude range: -90..90, longitude range: -180..180.

    Exceptions thrown
    IllegalArgumentException

    Thrown if arguments are null, or if the syntax of the mapcode is incorrect.

    UnknownMapcodeException

    Thrown if the mapcode has the right syntax, but cannot be decoded into a point.

    The accepted format is:

  8. def encode(latDeg: Double, lonDeg: Double, restrictToTerritory: Territory.Territory): Seq[Mapcode]

    Encode a lat/lon pair to a mapcode with territory information, for a specific territory.

    Encode a lat/lon pair to a mapcode with territory information, for a specific territory. This produces a potentially empty list of mapcodes (empty if the lat/lon does not fall within the territory for mapcodes).

    The returned result list will always contain at least 1 mapcode, because every lat/lon pair can be encoded.

    The list is ordered in such a way that the first result contains the shortest mapcode (which is usually a local mapcode).

    latDeg

    Latitude, accepted range: -90..90.

    lonDeg

    Longitude, accepted range: -180..180.

    restrictToTerritory

    Try to encode only within this territory, see { @link com.mapcode.Territory}. Cannot be null.

    returns

    List of mapcode information records, see { @link Mapcode}. This list is empty if no Mapcode can be generated for this territory matching the lat/lon.

    Exceptions thrown
    IllegalArgumentException

    Thrown if latitude or longitude are out of range.

  9. def encode(latDeg: Double, lonDeg: Double): Seq[Mapcode]

    Encode a lat/lon pair to a mapcode with territory information.

    Encode a lat/lon pair to a mapcode with territory information. This produces a non-empty list of mapcode, with at least 1 mapcode for the lat/lon, which is the "International" mapcode.

    The returned result list will always contain at least 1 mapcode, because every lat/lon pair can be encoded.

    The list is ordered in such a way that the first result contains the shortest mapcode (which is usually a local mapcode). The last result contains the "International" or world-wide mapcode, which is always unambiguous, even when used without a territory specification.

    The international code can be obtained from call like encode(lat, lon).last.

    latDeg

    Latitude, accepted range: -90..90.

    lonDeg

    Longitude, accepted range: -180..180.

    returns

    Non-empty, ordered list of mapcode information records, see { @link Mapcode}.
